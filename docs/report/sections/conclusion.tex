\section{Conclusion}

This project demonstrates the application of Answer Set Programming to optimize NHL 26 HUT line combinations. We developed a modular architecture that supports both batch optimization (Goal 1) and interactive optimization (Goal 2), separating data management, ASP encodings, solver integration, orchestration, and storage.

The system successfully implements:
\begin{itemize}
    \item A complete data model and SQLite-based storage for player cards and line combinations
    \item ASP rule encoding for abstract optimization (Goal 1 Stage A) with validated logical correctness
    \item \textbf{Goal 1 Stage A Integration}: \texttt{ClingoStageASolver} with Clingo execution, parsing, and symmetry breaking
    \item \textbf{Goal 1 Stage B Integration}: \texttt{ClingoStageBSolver} with concrete line enumeration, constraint enforcement, and deduplication
    \item A full Goal 1 pipeline orchestrator and result persistence workflow
    \item \textbf{Pipeline Manager}: asynchronous auto-run that executes the Goal 1 pipeline when results are missing without blocking the API event loop
    \item REST API endpoints for querying players, combos, and stored Goal 1 optimization results
    \item A full-featured React frontend with pages for dataset exploration and viewing Goal 1 best lines
    \item \textbf{Goal 2 Core/ASP Integration}: direct concrete line optimization implemented via a replicated generator--solver--pipeline pattern and validated through a dedicated test suite
\end{itemize}

Goal 2 introduces a direct, interactive optimization approach: instead of a two-stage abstraction, it forms concrete lines directly from candidate players, supports multiple optimization targets (OVR/SAL/AP and combined modes), and optionally enforces required combo IDs. This design matches the Goal 1 integration style while enabling lower-latency, interactive use.

\textbf{Remaining integration work}: while Goal 2 is implemented and tested in the backend Core/ASP layer, the project still requires wiring Goal 2 into the FastAPI optimization endpoints and connecting it to the frontend Optimize page for a complete end-to-end interactive user flow.

Future work should therefore focus on:
\begin{itemize}
    \item Completing Goal 2 API endpoint integration and frontend usage
    \item Implementing Goal 3 full-team optimization
    \item Profiling solver performance on large candidate pools and tuning ASP encodings
    \item Evaluating solution quality relative to heuristics and manual selections
\end{itemize}

Overall, the project illustrates how symbolic reasoning (ASP) and data-driven filtering (SQLite + candidate selection) can be combined into a scalable optimization system with strong correctness guarantees and modular extensibility.
